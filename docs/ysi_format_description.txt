The following is a complete desciption of the PC6000 and EW internal data file
format.  It has been excerpted from various source code files and source dox
files, and then cleaned up.  Additional info has been added.



Data File Format Notes

The data file format is described here for advanced users who wish to write
their own processing programs.	The data file uses a binary record-oriented
file format.  All values are Intel memory format (the LSB is in the lowest
memory location).  All floating point values are 4-byte IEEE format.  All
characters come from the full IBM PC character set.

The entire file is a series of records.  The first byte of each record is
the record type, which determines the length of the record.  There is no
end-of-record mark.  The overall file structure is as follows:

1 header record.
1 to 64 channel records.
0 or more delta, calibration, or datum records.
1 or more manual, logged, fixed, or dummy records.

The format of each record is described below using source code from the C
programming language.  The length of each data record is determined by the
number of channel records in the data file.  Specifically, there is one float
in each data record, for each channel record in the file.  Note, a byte is an
unsigned 1-byte value, a word is an unsigned 2-byte value, and a dword is an
unsigned 4-byte value.

Note if you are writing software that will ONLY be processing files created
by EW, you need only handle certain record types: one HEADER record, followed
by one or more CHANNEL records, followed by one or more LOGGED records.  The
present version of EW (and all past versions) have never created data files
containing any other record types, and there are no plans change this.	But
all Endeco/YSI software supports all of the record types below, and Endeco/YSI
reserves the right to use the other record types in future releases.



Data File Format C Language Definitions

typedef  unsigned char	byte;	/* handy data types /
typedef  unsigned int	word;
typedef  unsigned long	dword;

#define  MAXCHANS	32	/* max number of input channel records /

typedef struct {		 /* header record		 /
    byte   type;		/* record type = 'A'            /
    word   instr;		/* instrument type (decimal)	/
    dword  sig; 		/* system signature = 870489733 /
    word   progver;		/* program version times 100	/
    char   serno [16];		/* serial number (asciiz)	/
    char   sitename [32];	/* site id string (asciiz)	/
    char   pad1 [6];		/* future use (zeros)		/
    dword  interval;		/* logging interval (seconds)	/
    long   studytime;		/* julian time logging began	/
    long   firsttime;		/* julian time of first sample	/
    char   pad2 [36]; } 	/* future use (zeros)		/
    hed_t;

typedef struct {		/* channel record		/
    byte   type;		/* record type = 'B'            /
    int    schan;		/* sonde channel number 	/
    int    sen; 		/* sensor type			/
    word   probe;		/* probe type			/
    float  zero;		/* zero scale in ysi units	/
    float  full; }		/* full scale in ysi units	/
    chn_t;

typedef struct {		/* manual record		/
    byte   type;		/* record type = 'C'            /
    long   time;		/* time of reading (julian)	/
    float  data [MAXCHANS]; }	/* normalized data		/
    man_t;

typedef struct {		/* logged record		/
    byte   type;		/* record type = 'D'            /
    long   time;		/* time of reading (julian)	/
    float  data [MAXCHANS]; }	/* normalized data		/
    lgg_t;

typedef struct {		/* fixed record 		/
    byte   type;		/* record type = 'E'            /
    float  data [MAXCHANS]; }	/* normalized data		/
    fix_t;

typedef struct {		/* dummy record 		/
    byte   type; }		/* record type = 'F'            /
    dum_t;

typedef struct {		/* delta record 		/
    byte   type;		/* record type = 'G'            /
    long   time;		/* time of delta (julian)	/
    int    schan;		/* sonde channel number 	/
    int    sen; 		/* sensor type			/
    float  instr;		/* would display without offset /
    float  nbs; }		/* what user entered		/
    del_t;

typedef struct {		 /* cal record			 /
    byte   type;		/* record type = 'H'            /
    long   time;		/* time of cal (julian) 	/
    int    schan;		/* sonde channel number 	/
    int    sen; 		/* sensor type			/
    int    method;		/* calibration method = 1	/
    float  coeff [4]; } 	/* 4 calibration coeffs 	/
    cal_t;

typedef struct {		/* datum record 		/
    byte   type;		/* record type = 'I'            /
    long   time;		/* time of datum (julian)	/
    int    sen; 		/* sensor type			/
    float  nbs; }		/* what user entered		/
    dat_t;

Julian Time

To simplify the internal structure of the program, and to speed its operation,
all dates and times are represented in julian format.  The implementation of
julian format used here is defined as the number of seconds past midnight
March 1, 1984.	This format allows time differences to be calculated directly,
and time intervals to be added to dates directly.  It also allows dates and
times to be stored very compactly.  The conversion formulas below are valid
for dates in 1985 through 2051.  The conversion formulas are as follows:

Calendar to julian conversion:
if (year < 100) year = 1984 + (year + 16) modulo 100;
if (month < 3) { month = month + 12; year = year - 1; }
date = int (365.25 * year) + int (30.6 * month + day + .7) - 724749;
julian = 86400 * date + 3600 * hour + 60 * minute + second;

Julian to calendar conversion:
date = julian / 86400;
time = julian modulo 86400;
hour = time / 3600;
minute = (time modulo 3600) / 60;
second = time modulo 60;
year = (date + .8) / 365.25;
date = date - int (year * 365.25);
month = (date + .5) / 30.6;
day = date - int (30.6 * month + .5) + 1;
month = month + 3;
if (month > 12) { month = month - 12; year = year + 1; }
year = (year + 1984) modulo 100;
